# Publishing Workflow

Publishing a component to the SCM registry is done exclusively through the CLI tool with automated validation and auto-merge workflows.

## Publishing Steps

<Steps>
  <Step title="Create Component">
    Use `scm create` to generate component structure
  </Step>
  <Step title="Develop & Test">
    Customize component and test thoroughly
  </Step>
  <Step title="Publish">
    Run `scm publish` to submit to registry
  </Step>
  <Step title="Automated Validation">
    GitHub Actions validate structure and metadata
  </Step>
  <Step title="Auto-Merge">
    Valid components are automatically merged
  </Step>
  <Step title="Available">
    Component appears in registry index
  </Step>
</Steps>

## CLI Publishing Commands

<CodeGroup>
  <CodeGroupItem title="Basic Publish">
    ```bash
    scm publish
    ```
    Standard publish with confirmation prompts
  </CodeGroupItem>
  <CodeGroupItem title="Skip Confirmation">
    ```bash
    scm publish --yes
    ```
    Skip confirmation prompts
  </CodeGroupItem>
  <CodeGroupItem title="Custom Message">
    ```bash
    scm publish --message "feat: add new button variant"
    ```
    Use custom commit message
  </CodeGroupItem>
</CodeGroup>

## Automated Validation

<CardGroup cols={2}>
  <Card title="Required Fields">
    - name, type, title, description
    - Valid component type
    - Semantic versioning
  </Card>
  <Card title="File Validation">
    - All files must exist
    - Proper file organization
    - Valid file paths
  </Card>
  <Card title="Naming Rules">
    - Lowercase with hyphens only
    - No reserved names
    - Valid characters: a-z, 0-9, -
  </Card>
  <Card title="Component Types">
    - registry:component
    - registry:block
    - registry:hook
    - registry:lib
  </Card>
</CardGroup>

## Common Validation Errors

<CardGroup cols={2}>
  <Card title="Missing Fields">
    **Error:** Missing required field: name/type/title/description
    
    **Solution:** Ensure all required fields are present in registry.json
  </Card>
  <Card title="Invalid Type">
    **Error:** Invalid type: registry:ui. Must be one of: registry:component, registry:block, registry:hook, registry:lib
    
    **Solution:** Use correct component type
  </Card>
  <Card title="File Not Found">
    **Error:** File not found: registry/components/MyComponent.tsx
    
    **Solution:** Ensure all files listed in registry.json exist
  </Card>
  <Card title="Reserved Name">
    **Error:** Component name 'button' is reserved and cannot be used
    
    **Solution:** Choose a different component name
  </Card>
</CardGroup>

## Troubleshooting

<CardGroup cols={2}>
  <Card title="Validation Failed">
    **Issue:** Component validation failed
    
    **Solutions:**
    - Check error messages for missing fields
    - Fix issues in registry.json or component files
    - Ensure all required files exist
    - Verify file paths are correct
  </Card>
  <Card title="Permission Denied">
    **Issue:** Permission denied during publish
    
    **Solutions:**
    - Ensure you are authenticated (`scm login`)
    - Check GitHub permissions
    - Verify repository access
    - Check token scopes
  </Card>
  <Card title="Component Not Showing">
    **Issue:** Component not appearing after merge
    
    **Solutions:**
    - Wait for registry index update
    - Check for build errors
    - Verify component structure
    - Contact maintainers if needed
  </Card>
</CardGroup>
