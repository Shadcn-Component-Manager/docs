## Step-by-Step Process

<Steps>
  <Step title="Install CLI">
    Install the SCM CLI tool globally
  </Step>
  <Step title="Authenticate">
    Run `scm login` to authenticate with GitHub
  </Step>
  <Step title="Create Component">
    Use `scm create` to generate component structure
  </Step>
  <Step title="Develop & Test">
    Customize component and test thoroughly
  </Step>
  <Step title="Publish">
    Run `scm publish` to submit to registry
  </Step>
  <Step title="Automated Validation">
    GitHub Actions validate structure and metadata
  </Step>
  <Step title="Auto-Merge">
    Valid components are automatically merged
  </Step>
  <Step title="Available">
    Component appears in registry index
  </Step>
</Steps>

## Detailed Workflow

### Step 1: Install and Authenticate

<CodeGroup>
  <CodeGroupItem title="Install CLI">
    ```bash
    npm install -g @shadcn-component-manager/scm
    ```
  </CodeGroupItem>
  <CodeGroupItem title="Authenticate">
    ```bash
    scm login
    ```
    Follow the prompts to authenticate with GitHub
  </CodeGroupItem>
</CodeGroup>

### Step 2: Create Component

```bash
scm create my-awesome-component
```

This generates:
- Proper component structure
- `registry.json` template
- Component files in `registry/` subdirectories

### Step 3: Develop Your Component

<Steps>
  <Step title="Edit Files">
    Customize the generated component files
    Update metadata in `registry.json`
    Add your component logic
  </Step>
  <Step title="Test Locally">
    Test your component in a clean project
    Ensure all functionality works
    Check for TypeScript errors
  </Step>
  <Step title="Update Metadata">
    Update version, description, dependencies
    Add appropriate categories
    Ensure all required fields are present
  </Step>
</Steps>

### Step 4: Publish Component

```bash
scm publish
```

This will:
- Validate your component structure
- Create a pull request to the registry
- Trigger automated validation
- Auto-merge if validation passes

## Common Issues and Solutions

<CardGroup cols={2}>
  <Card title="Validation Failures">
    **Issue:** Component validation failed
    
    **Solutions:**
    - Check error messages for missing fields
    - Fix issues in registry.json or component files
    - Ensure all required files exist
    - Verify file paths are correct
  </Card>
  <Card title="Permission Issues">
    **Issue:** Permission denied during publish
    
    **Solutions:**
    - Ensure you are authenticated (`scm login`)
    - Check GitHub permissions
    - Verify repository access
    - Check token scopes
  </Card>
  <Card title="Reserved Names">
    **Issue:** Component name is reserved
    
    **Solutions:**
    - Choose a different component name
    - Check the reserved names list
    - Use descriptive, unique names
  </Card>
  <Card title="File Organization">
    **Issue:** Files not found or improperly organized
    
    **Solutions:**
    - Use `scm create` for proper structure
    - Organize files in registry/ subdirectories
    - Ensure all paths in registry.json are correct
  </Card>
</CardGroup>
