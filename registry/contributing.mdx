## How to Contribute

<Steps>
  <Step title="Install CLI">
    Install the SCM CLI tool
  </Step>
  <Step title="Authenticate">
    Run `scm login` to authenticate with GitHub
  </Step>
  <Step title="Create Component">
    Use `scm create` to generate component structure
  </Step>
  <Step title="Develop & Test">
    Customize component and test thoroughly
  </Step>
  <Step title="Publish">
    Run `scm publish` to submit to registry
  </Step>
  <Step title="Auto-Merge">
    Valid components are automatically merged
  </Step>
</Steps>

<CardGroup cols={2}>

## Quality Standards

<CardGroup cols={2}>
  <Card title="Code Quality">
    - Use TypeScript for all components
    - Follow component patterns
    - Keep code modular and maintainable
    - Use descriptive names
  </Card>
  <Card title="Documentation">
    - Write clear component descriptions
    - Document all props and variants
    - Include usage examples
    - Test thoroughly before publishing
  </Card>
  <Card title="Testing">
    - Test in clean projects
    - Ensure compatibility
    - Check accessibility
    - Fix all lint errors
  </Card>
  <Card title="Dependencies">
    - Keep dependencies minimal
    - Use semantic versioning
    - Avoid breaking changes
    - Test with all dependencies
  </Card>
</CardGroup>

## Validation Requirements

<CardGroup cols={2}>
  <Card title="Required Fields">
    - name, type, title, description
    - Valid component type
    - Semantic versioning
  </Card>
  <Card title="File Structure">
    - All files must exist
    - Proper file organization
    - Valid file paths
  </Card>
  <Card title="Naming Rules">
    - Lowercase with hyphens only
    - No reserved names
    - Valid characters: a-z, 0-9, -
  </Card>
  <Card title="Component Types">
    - registry:component
    - registry:block
    - registry:hook
    - registry:lib
  </Card>
</CardGroup>

## Pre-Submission Checklist

<CardGroup cols={2}>
  <Card title="Component Quality">
    - [ ] Component works in clean project
    - [ ] All TypeScript errors resolved
    - [ ] Linting passes without errors
    - [ ] Documentation is complete
  </Card>
  <Card title="Metadata">
    - [ ] `registry.json` is valid and complete
    - [ ] Version is bumped appropriately
    - [ ] Dependencies are correctly listed
    - [ ] Categories are appropriate
  </Card>
  <Card title="Files">
    - [ ] All files exist and are valid
    - [ ] Files are organized in registry/ subdirectories
    - [ ] Component file has correct extension
    - [ ] Examples are working
  </Card>
  <Card title="Testing">
    - [ ] Tested in multiple environments
    - [ ] Accessibility audit completed
    - [ ] Performance tested
    - [ ] Error states handled
  </Card>
</CardGroup>

## Common Issues

<CardGroup cols={2}>
  <Card title="Validation Failures">
    **Issue:** Component validation failed
    
    **Solutions:**
    - Check error messages for missing fields
    - Fix issues in registry.json or component files
    - Ensure all required files exist
    - Verify file paths are correct
  </Card>
  <Card title="Permission Issues">
    **Issue:** Permission denied during publish
    
    **Solutions:**
    - Ensure you are authenticated (`scm login`)
    - Check GitHub permissions
    - Verify repository access
    - Check token scopes
  </Card>
  <Card title="Reserved Names">
    **Issue:** Component name is reserved
    
    **Solutions:**
    - Choose a different component name
    - Check the reserved names list
    - Use descriptive, unique names
  </Card>
  <Card title="File Organization">
    **Issue:** Files not found or improperly organized
    
    **Solutions:**
    - Use `scm create` for proper structure
    - Organize files in registry/ subdirectories
    - Ensure all paths in registry.json are correct
  </Card>
</CardGroup>
