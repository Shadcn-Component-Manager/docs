# Registry Overview

The SCM registry is the central hub for sharing, discovering, and managing components. It's a GitHub-based repository that powers the entire SCM ecosystem through automated workflows and CLI integration.

## What is the SCM Registry?

The registry is a public, open-source repository that stores all community-contributed components with automated validation, indexing, and publishing workflows.

## Registry Structure

```
registry/
├── registry.json                    # Auto-generated index
├── components/                      # Component storage
│   ├── {github-username}/          # User namespace
│   │   ├── {component-name}/       # Component directory
│   │   │   ├── 1.0.0/             # Version directory
│   │   │   │   ├── registry.json  # Component metadata
│   │   │   │   └── registry/      # Component files
│   │   │   └── 2.0.0/             # Newer version
└── .github/workflows/              # GitHub Actions
```

## Validation Requirements

## Registry Index

The `registry.json` file is automatically generated and contains:

```json
[
  {
    "name": "username/component",
    "description": "Component description",
    "author": "Author Name",
    "categories": ["ui", "button"],
    "version": "1.0.0",
    "lastUpdated": "2024-01-15T10:30:00.000Z"
  }
]
```
