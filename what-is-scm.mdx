# What is SCM?

SCM (Shadcn Component Manager) is an open-source CLI tool and registry ecosystem designed to extend shadcn's component model, enabling developers to create, share, and install UI components with ease.

<Note>
  SCM is inspired by the NPM registry concept but specifically built for React components and the shadcn/ui ecosystem.
</Note>

## The Problem

<CardGroup cols={2}>
  <Card title="Discovery">
    Finding quality shadcn/ui components is difficult
  </Card>
  <Card title="Installation">
    Complex dependency management and error-prone setup
  </Card>
  <Card title="Quality">
    Inconsistent component quality and documentation
  </Card>
  <Card title="Management">
    No centralized component discovery for shadcn ecosystem
  </Card>
</CardGroup>

## The Solution

SCM provides a complete ecosystem:

<CardGroup cols={2}>
  <Card title="CLI Tool" icon="terminal">
    Install and manage components with automatic dependency resolution
  </Card>
  <Card title="Registry" icon="database">
    Centralized GitHub-based component repository
  </Card>
  <Card title="Web Interface" icon="globe">
    Modern Next.js app for browsing and previewing components
  </Card>
  <Card title="Quality Control" icon="shield-check">
    Automated validation and human review process
  </Card>
</CardGroup>

## How It Works

<Steps>
  <Step title="Create">
    Developers create components using `scm create`
  </Step>
  <Step title="Publish">
    Submit to registry via GitHub pull requests
  </Step>
  <Step title="Review">
    Automated checks and human review ensure quality
  </Step>
  <Step title="Discover">
    Users search and browse components via CLI or web
  </Step>
  <Step title="Install">
    One-command installation with automatic dependency management
  </Step>
</Steps>
