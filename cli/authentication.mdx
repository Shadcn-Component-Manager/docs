# Authentication

SCM uses GitHub OAuth device flow for secure authentication.

## Login

```bash
scm login
```

<Note>
  **What happens:**
  - Opens browser for GitHub authorization
  - Requests necessary permissions (repo, read:user, user:email)
  - Stores token securely in `~/.scm/config.json`
</Note>

## Login Options

```bash
# Force re-authentication
scm login --force

# Use existing token instead of OAuth flow
scm login --token your-github-token

# Check current authentication status
scm login --check

# Show detailed auth info
scm login --verbose
```

## Logout

```bash
scm logout
```

Removes stored authentication token.

## Logout Options

```bash
# Clear all cached data (not just token)
scm logout --all

# Clear cache only
scm logout --cache

# Force logout without confirmation
scm logout --force
```

## OAuth Flow

<Steps>
  <Step title="Request Authorization">
    SCM generates device authorization code
  </Step>
  <Step title="Browser Authorization">
    Opens browser to GitHub authorization page
  </Step>
  <Step title="Token Exchange">
    Exchanges authorization code for access token
  </Step>
  <Step title="Store Token">
    Securely stores token for future use
  </Step>
</Steps>
