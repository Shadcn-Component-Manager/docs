# Publish Command

Publish a component to the SCM registry with automatic versioning.

## Basic Usage

```bash
scm publish
```

Run this command from your component directory (where `registry.json` is located).

## Options

```bash
# Skip confirmation prompts
scm publish --yes

# Custom commit message
scm publish --message "feat: add new button variant"

# Specific item from registry collection
scm publish --item my-component

# Specify version manually (override auto-detection)
scm publish --version 2.0.0
```

## Publishing Workflow

<Steps>
  <Step title="Validate">
    Validates component structure and metadata
  </Step>
  <Step title="Check Conflicts">
    Checks for reserved names and conflicts
  </Step>
  <Step title="Detect Version">
    Automatically detects version changes
  </Step>
  <Step title="Create PR">
    Creates GitHub branch and pull request
  </Step>
  <Step title="Review">
    Registry maintainers review and merge
  </Step>
</Steps>

## Component Requirements

### Valid registry.json

```json
{
  "name": "my-component",
  "type": "registry:ui",
  "title": "My Component",
  "description": "A description of your component",
  "files": [
    {
      "path": "MyComponent.tsx",
      "type": "registry:ui"
    }
  ],
  "dependencies": ["react@^18.0.0"],
  "version": "1.0.0",
  "author": "Your Name <your-email@example.com>",
  "categories": ["ui"]
}
```

## GitHub Integration

### Pull Request Creation

When you publish, SCM creates a pull request:

- **Branch Name**: `component/username/component-name-version`
- **Commit Message**: `feat: add component-name vversion`
- **PR Title**: `Add username/component-name vversion`

### Review Process

<Steps>
  <Step title="Automated Checks">
    GitHub Actions validate the component
  </Step>
  <Step title="Maintainer Review">
    Registry maintainers review the PR
  </Step>
  <Step title="Feedback">
    You may receive feedback or requests for changes
  </Step>
  <Step title="Merge">
    Once approved, the PR is merged
  </Step>
</Steps>
