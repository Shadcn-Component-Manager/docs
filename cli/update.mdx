# Update Components

Update installed components to their latest versions.

## Basic Usage

```bash
# Update all components
scm update

# Update specific component
scm update username/component-name

# Check for updates (dry run)
scm update --dry-run
```

## Options

```bash
# Force update even if no newer version available
scm update --force

# Show what would be updated without making changes
scm update --dry-run

# Update to specific version (instead of latest)
scm update --version 2.0.0

# Update all components (explicit flag)
scm update --all

# Skip updating dependencies
scm update --skip-deps

# Only check for updates, don't install
scm update --check-only

# Interactive mode for each component
scm update --interactive
```

## Examples

```bash
# Update all components
scm update

# Update specific component
scm update johndoe/awesome-button

# Check what would be updated
scm update --dry-run

# Force update even if no newer version
scm update --force

# Update to specific version
scm update johndoe/awesome-button --version 2.0.0

# Interactive mode
scm update --interactive

# Check only (no updates)
scm update --check-only
```

## What Happens

<Steps>
  <Step title="Check Versions">
    Compares installed versions with latest available
  </Step>
  <Step title="Download Updates">
    Downloads updated component files
  </Step>
  <Step title="Install Dependencies">
    Installs new dependencies if needed
  </Step>
  <Step title="Update Files">
    Replaces old files with updated versions
  </Step>
  <Step title="Track Changes">
    Updates tracking information
  </Step>
</Steps>

## Version Tracking

SCM tracks installed components in `.scm-installed.json`:

```json
{
  "johndoe/awesome-button": {
    "version": "1.1.0",
    "installedAt": "2024-01-15T10:30:00.000Z"
  }
}
```

## Interactive Mode

When using `--interactive`, SCM prompts for each component:

```bash
$ scm update --interactive
Update johndoe/awesome-button from 1.0.0 to 1.1.0? (Y/n)
```

## Troubleshooting

<CardGroup cols={2}>
  <Card title="Update Fails">
    **Issue:** Update fails with errors
    
    **Solutions:**
    - Clear cache: `scm add component --force`
    - Check dependencies: `npm install`
    - Verify component exists: `scm preview component`
  </Card>
  <Card title="Version Conflicts">
    **Issue:** Version conflicts with dependencies
    
    **Solutions:**
    - Update dependencies: `npm update`
    - Check compatibility
    - Contact component author
  </Card>
</CardGroup>
